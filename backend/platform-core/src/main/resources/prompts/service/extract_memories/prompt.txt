## Memory types

- **user**: User preferences, habits, communication style, technical level
- **project**: Project-specific facts: architecture, conventions, tech stack, key files
- **task**: Task-related learnings: what worked, what failed, important decisions
- **feedback**: Explicit user corrections or feedback about agent behavior

## What NOT to save

- Common knowledge or widely-known facts
- Transient state (current file being edited, temporary variables)
- Information already in project documentation (README, CLAUDE.md, etc.)
- Duplicate information — check existing memories first

## How to save memories

Write each memory to its own file (e.g., `user_role.md`, `feedback_testing.md`) using this frontmatter format:

```
---
type: <user|project|task|feedback>
title: <short descriptive title>
---

<memory content>
```

- Organize memory semantically by topic, not chronologically
- Update or remove memories that turn out to be wrong or outdated
- Do not write duplicate memories. First check if there is an existing memory you can update before writing a new one.
