## 정책 상세

### POL-001 | 세션 TTL
- **적용 모듈**: 세션 (SessionStore)
- **정책 설명**: Redis에 저장되는 세션 메시지의 만료 시간. 기본값 24시간.
- **현재 값**: 24h (86400초)
- **변경 방법**: application.yml 설정 변경
- **관련 기능 ID**: PRD-089

### POL-002 | LLM 모델 기본값
- **적용 모듈**: LLM 어댑터
- **정책 설명**: 기본 채팅/임베딩 모델 설정
- **현재 값**:
  - Anthropic 채팅: `claude-sonnet-4-5`
  - OpenAI 채팅: `gpt-4o`
  - 임베딩: `BAAI/bge-m3` (기본), `text-embedding-3-small` (옵션)
- **변경 방법**: application.yml `platform.llm.default-model` 변경
- **관련 기능 ID**: PRD-001, PRD-002

### POL-003 | 임베딩 모델/차원
- **적용 모듈**: RAG (EmbeddingService)
- **정책 설명**: 임베딩 생성 시 사용하는 모델과 벡터 차원 수
- **현재 값**: model=BAAI/bge-m3, dimensions=1024 (환경변수 `DEFAULT_EMBED_MODEL`로 변경 가능. OpenAI text-embedding-3-small 1536d, KoSimCSE 768d 옵션)
- **변경 방법**: application.yml 설정 변경
- **관련 기능 ID**: PRD-052, PRD-053

### POL-004 | 서버 포트
- **적용 모듈**: 인프라
- **정책 설명**: Spring Boot 서버 실행 포트
- **현재 값**: 8080
- **변경 방법**: 환경변수 `SERVER_PORT`
- **관련 기능 ID**: 전체

### POL-005 | DB 연결 정보
- **적용 모듈**: 인프라
- **정책 설명**: Master/Tenant DB 연결 정보
- **현재 값**: localhost:5432 / platform:platform
- **변경 방법**: 환경변수 `MASTER_DB_URL`, `TENANT_DB_HOST` 등
- **관련 기능 ID**: 전체

### POL-006 | 테넌트 기본 구독 쿼터
- **적용 모듈**: 플랫폼관리
- **정책 설명**: 새 테넌트 생성 시 적용되는 기본 쿼터
- **현재 값**:
  - free: 월 100만 토큰, 연결 5개, 지식소스 3개, 워크플로우 10개, 스토리지 1GB, 사용자 5명
  - starter/pro/enterprise: 상위 플랜
- **변경 방법**: DB subscriptions 테이블 직접 수정 또는 API (PRD-087)
- **관련 기능 ID**: PRD-080, PRD-087

### POL-007 | 정책 규칙
- **적용 모듈**: 정책 (PolicyEngine)
- **정책 설명**: 런타임에 동적으로 적용되는 정책 규칙. 테넌트별 독립 관리.
- **규칙 유형**:
  - `DENY`: 즉시 거부
  - `REQUIRE_APPROVAL`: 승인 대기
  - `RATE_LIMIT`: 빈도 제한 (config: window, limit)
  - `TRANSFORM`: PII 마스킹 등 변환
  - `LOG`: 감사 로깅
  - `ALLOW`: 명시적 허용
- **변경 방법**: DB policies 테이블 또는 API (PRD-020~026)
- **관련 기능 ID**: PRD-020~026, PRD-095

### POL-008 | 라우팅 전략 및 규칙
- **적용 모듈**: 라우팅 (ModelRouter)
- **정책 설명**: LLM 모델 요청을 어느 프로바이더로 라우팅할지 결정하는 규칙
- **전략 유형**: round-robin, cost-optimized, latency
- **변경 방법**: DB routing_config 테이블 또는 API (PRD-033~037)
- **관련 기능 ID**: PRD-033~037

### POL-009 | 워크플로우 재시도 설정
- **적용 모듈**: 워크플로우 (WorkflowEngine)
- **정책 설명**: 워크플로우 스텝 실패 시 재시도 횟수와 지연 시간
- **현재 기본값**: retryMaxAttempts=3, retryDelayMs=1000 (지수 백오프)
- **변경 방법**: 워크플로우 생성/수정 시 errorHandling 필드로 설정
- **관련 기능 ID**: PRD-039, PRD-043

### POL-010 | 지식소스 청킹 설정
- **적용 모듈**: RAG (DocumentChunker)
- **정책 설명**: 문서를 청크로 분할하는 전략, 크기, 오버랩 설정
- **현재 기본값**: strategy=fixed, size=512, overlap=50
- **변경 방법**: 지식소스 생성/수정 시 chunkingConfig 필드로 설정
- **관련 기능 ID**: PRD-048

### POL-011 | 검색 설정
- **적용 모듈**: RAG (VectorSearcher)
- **정책 설명**: 벡터 검색 시 topK, 유사도 임계값, 검색 유형 등
- **현재 기본값**: topK=5, similarityThreshold=0.7, searchType=hybrid
- **변경 방법**: DB retrieval_config 테이블 또는 API (PRD-055~058)
- **관련 기능 ID**: PRD-055~058

### POL-012 | RBAC 권한 매핑
- **적용 모듈**: 사용자/역할
- **정책 설명**: 역할별 권한 목록. 16개 권한을 역할에 매핑.
- **권한 목록**: chat:read/write, connections:read/write, policies:read/write, workflows:read/write, schemas:read/write, prompts:read/write, knowledge:read/write, admin:read/write
- **변경 방법**: DB roles 테이블 또는 API (PRD-072~076)
- **관련 기능 ID**: PRD-072~076

### POL-016 | 대시보드 리프레시 간격
- **적용 모듈**: FE 대시보드
- **정책 설명**: 프론트엔드 각 데이터 소스의 자동 갱신 주기
- **현재 값**: 대시보드 통계 30초, 액션 로그 5초, 승인 대기 10초
- **변경 방법**: hooks 소스코드 수정 (refetchInterval)
- **관련 기능 ID**: PRD-059

### POL-017 | 도구 선택 제어 (tool_choice)
- **적용 모듈**: 오케스트레이터 (OrchestratorEngine)
- **정책 설명**: LLM 호출 시 도구 사용 전략을 제어하는 파라미터. API 요청별로 동적 지정 가능.
- **현재 값**: auto (기본)
- **옵션**: auto (LLM 자율 결정), none (도구 미사용), required (반드시 도구 사용), {tool_name} (특정 도구 강제)
- **변경 방법**: ChatRequest API 파라미터 `tool_choice` 지정, 워크플로우 LLM_CALL 스텝의 config.tool_choice 필드
- **관련 기능 ID**: PRD-097
- **관련 CR**: CR-006

### POL-018 | 도구 필터 컨텍스트
- **적용 모듈**: 오케스트레이터 (ToolRegistry)
- **정책 설명**: LLM에 노출할 도구 후보를 필터링하는 컨텍스트. 허용 목록, 차단 목록, 태그 기반 필터링 지원.
- **현재 값**: null (전체 노출)
- **변경 방법**: ChatRequest API 파라미터 또는 워크플로우 LLM_CALL 스텝의 config.allowed_tools / config.blocked_tools 필드
- **관련 기능 ID**: PRD-096
- **관련 CR**: CR-006

### POL-019 | 구조화 출력 응답 포맷
- **적용 모듈**: 오케스트레이터 (OrchestratorEngine)
- **정책 설명**: LLM 응답을 JSON Schema 기반 구조화 포맷으로 요청하는 설정. 인라인 스키마 또는 등록된 스키마 참조 지원.
- **현재 값**: 미지정 (텍스트 응답)
- **변경 방법**: ChatRequest API 파라미터 `response_format` 지정, 워크플로우 LLM_CALL 스텝의 config.response_schema 필드
- **관련 기능 ID**: PRD-098
- **관련 CR**: CR-007

### POL-020 | 정책 규칙 타입 템플릿
- **적용 모듈**: 정책 FE (Policies 페이지)
- **정책 설명**: 정책 생성 시 규칙 타입을 선택하면 자동 생성되는 JSON 템플릿. 6가지 규칙 타입별 기본 구조 제공.
- **현재 값**: content_filter, cost_limit, token_limit, rate_limit, model_filter, time_restriction
- **변경 방법**: FE 소스코드 수정 (generateRuleTemplate 함수)
- **관련 기능 ID**: PRD-128
- **관련 CR**: CR-015

### POL-021 | RAG 평가 모드
- **적용 모듈**: RAG (evaluate_rag API)
- **정책 설명**: RAG 평가 실행 시 사용할 평가 방식. fast(임베딩 유사도) 또는 accurate(LLM Judge).
- **현재 값**: fast (기본)
- **변경 방법**: evaluate_rag API 파라미터 `mode` 지정, FE RagEvaluation 페이지 라디오 버튼
- **관련 기능 ID**: PRD-130
- **관련 CR**: CR-016

### POL-022 | LLM Judge 모델
- **적용 모듈**: Python-평가 (evaluate_rag_quality MCP 도구)
- **정책 설명**: LLM Judge 평가 시 사용하는 LLM 모델 설정.
- **현재 값**: Claude Haiku (claude-3-haiku-20240307)
- **변경 방법**: 환경변수 `LLM_JUDGE_MODEL` 변경
- **관련 기능 ID**: PRD-130
- **관련 CR**: CR-016

### POL-023 | ConfigPanel 스키마 프리셋
- **적용 모듈**: FE 워크플로우 스튜디오 (ConfigPanel)
- **정책 설명**: LLM_CALL 노드의 response_schema 필드에서 사용할 수 있는 스키마 프리셋 목록.
- **현재 값**: 분류(classification), 요약(summary), 추출(extraction) 3가지
- **변경 방법**: FE 소스코드 수정 (프리셋 목록 추가)
- **관련 기능 ID**: PRD-133
- **관련 CR**: CR-017

## 정책 vs 비즈니스 규칙 구분

| 구분 | 정책 (T1-4) | 비즈니스 규칙 (T1-3) |
|------|------------|---------------------|
| 성격 | 운영 중 변경 가능 | 불변, 코드 고정 |
| 저장 | DB / 설정파일 / 환경변수 | 소스코드 하드코딩 |
| 예시 | 세션 TTL 24시간 | 도구 호출 최대 5회 |
| 변경 주체 | 운영자/관리자 | 개발자 (코드 배포) |
