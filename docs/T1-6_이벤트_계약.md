> 설계 버전: 2.0 | 최종 수정: 2026-04-07 | 관련 CR: CR-001, CR-031

> **프로젝트**: Aimbase
> **작성일**: 2026-03-10 (역설계)
> **비고**: 본 시스템은 동기 REST 기반이며, 비동기 이벤트 버스(Kafka 등)는 미사용. 내부 이벤트는 메서드 호출, 외부 이벤트는 Webhook/WebSocket으로 처리.

## 발행 이벤트 (내부)

### SessionMessageAppended (v1)
- **트리거 조건**: 채팅 완료 시 세션에 사용자/어시스턴트 메시지 저장
- **주요 포함 데이터**: sessionId, tenantId, role, contentBlocks[], timestamp
- **소비 주체**: SessionStore (Redis 저장)
- **관련 기능 ID**: PRD-089

### LLMCallCompleted (v1)
- **트리거 조건**: LLM 어댑터가 응답을 반환한 시점
- **주요 포함 데이터**: requestId, model, provider, inputTokens, outputTokens, costUsd, latencyMs, finishReason
- **소비 주체**: UsageLogRepository (사용량 기록), PlatformMetrics (Prometheus), AuditLogger (감사 로그)
- **관련 기능 ID**: PRD-001, PRD-002

### ToolCallExecuted (v1)
- **트리거 조건**: ToolCallHandler에서 도구 실행 완료
- **주요 포함 데이터**: toolName, input, output, durationMs, success, iterationNumber
- **소비 주체**: AuditLogger
- **관련 기능 ID**: PRD-092

### ActionExecuted (v1)
- **트리거 조건**: Write/Notify 액션 실행 완료
- **주요 포함 데이터**: actionId, type(WRITE/NOTIFY), adapter, destination, status, resultSummary
- **소비 주체**: ActionLogRepository, AuditLogger
- **관련 기능 ID**: PRD-093, PRD-094

### PolicyEvaluated (v1)
- **트리거 조건**: PolicyEngine에서 정책 평가 완료
- **주요 포함 데이터**: policyIds[], action(ALLOW/DENY/REQUIRE_APPROVAL), intent, triggeredRules[], transforms
- **소비 주체**: AuditLogger
- **관련 기능 ID**: PRD-095

### ApprovalRequested (v1)
- **트리거 조건**: HUMAN_INPUT 스텝 도달 또는 REQUIRE_APPROVAL 정책 트리거
- **주요 포함 데이터**: approvalId, workflowRunId/actionId, approvers[], channel, requestedAt, timeoutAt
- **소비 주체**: PendingApprovalRepository, WebSocketAdapter (실시간 알림)
- **관련 기능 ID**: PRD-046, PRD-095

### ApprovalResolved (v1)
- **트리거 조건**: 관리자가 승인 또는 거부 처리
- **주요 포함 데이터**: approvalId, status(approved/rejected), approvedBy, reason, resolvedAt
- **소비 주체**: WorkflowEngine.resume() (워크플로우 재개)
- **관련 기능 ID**: PRD-064, PRD-065

### WorkflowRunStarted (v1)
- **트리거 조건**: POST /workflows/{id}/run 호출 시
- **주요 포함 데이터**: workflowId, runId, sessionId, inputData, startedAt
- **소비 주체**: WorkflowRunRepository
- **관련 기능 ID**: PRD-043

### WorkflowRunCompleted (v1)
- **트리거 조건**: 모든 스텝 정상 완료
- **주요 포함 데이터**: workflowId, runId, stepResults, completedAt, durationMs
- **소비 주체**: WorkflowRunRepository
- **관련 기능 ID**: PRD-043

### WorkflowRunFailed (v1)
- **트리거 조건**: 스텝 실패 (재시도 소진) 또는 승인 거부
- **주요 포함 데이터**: workflowId, runId, failedStep, error, failedAt
- **소비 주체**: WorkflowRunRepository
- **관련 기능 ID**: PRD-043

### TenantProvisioned (v1)
- **트리거 조건**: 테넌트 프로비저닝 6단계 완료
- **주요 포함 데이터**: tenantId, name, dbName, plan, adminEmail, provisionedAt
- **소비 주체**: TenantDataSourceManager (DataSource 캐시 등록)
- **관련 기능 ID**: PRD-080

### IngestionCompleted (v1)
- **트리거 조건**: 인제스션 파이프라인 정상 완료
- **주요 포함 데이터**: sourceId, documentCount, chunkCount, durationMs, completedAt
- **소비 주체**: KnowledgeSourceRepository (status=completed, counts 갱신), IngestionLogRepository
- **관련 기능 ID**: PRD-052

### MCPToolsDiscovered (v1)
- **트리거 조건**: MCP 서버 도구 탐색 완료
- **주요 포함 데이터**: serverId, toolCount, tools[{name, description, inputSchema}]
- **소비 주체**: ToolRegistry (동적 등록), MCPServerRepository (toolsCache 갱신)
- **관련 기능 ID**: PRD-013

## 발행 이벤트 (외부)

### WebSocketNotification (v1)
- **트리거 조건**: NOTIFY 액션에서 WebSocket 어댑터 호출
- **주요 포함 데이터**: topic, message, metadata, timestamp
- **소비 주체**: 프론트엔드 WebSocket 클라이언트
- **관련 기능 ID**: PRD-094

### SlackNotification (v1)
- **트리거 조건**: NOTIFY 액션에서 Slack 어댑터 호출
- **주요 포함 데이터**: channel, text, blocks[], botToken
- **소비 주체**: Slack API (외부 서비스)
- **관련 기능 ID**: PRD-094
- **정합성 전략**: fire-and-forget (재시도 미지원)

## 수신 이벤트 (외부)

### MCPToolCall / MCPToolResult (v1)
- **트리거 조건**: MCP 프로토콜을 통한 도구 호출 요청/응답
- **주요 포함 데이터**: toolName, input/output, callId
- **발행 주체**: MCP Server (외부)
- **소비 주체**: MCPServerClient → MCPToolExecutor
- **관련 기능 ID**: PRD-013

### LLMProviderResponse (v1)
- **트리거 조건**: LLM 프로바이더(Anthropic/OpenAI/Ollama)로부터 응답 수신
- **주요 포함 데이터**: provider-specific response (messages, tool_calls, usage)
- **발행 주체**: Anthropic API / OpenAI API / Ollama API
- **소비 주체**: LLMAdapter (변환 → LLMResponse)
- **관련 기능 ID**: PRD-001, PRD-002
- **멱등 키**: requestId (LLM 요청별 고유 ID)

---

### PostCompactRecoveryCompleted (v1) [CR-031]
- **트리거 조건**: ContextWindowManager 압축 후 PostCompactRecoveryService가 컨텍스트 복구를 완료한 시점
- **주요 포함 데이터**: sessionId, compactionLevel, restoredFileCount, restoredMemoryCount, totalTokensRestored, durationMs
- **소비 주체**: AuditLogger, 모니터링 (복구 빈도/비용 추적)
- **관련 기능 ID**: PRD-211

### MemoryAutoExtracted (v1) [CR-031]
- **트리거 조건**: SESSION_END 훅 후 메모리 자동 추출이 완료된 시점
- **주요 포함 데이터**: sessionId, userId, extractedCount, duplicateCount, newCount, memoryTypes[]
- **소비 주체**: AuditLogger, ConversationMemoryRepository
- **관련 기능 ID**: PRD-213

## 설계 철학

1. **동기 우선 아키텍처**: 현재 시스템은 REST 동기 호출 중심. 비동기 이벤트 버스(Kafka/RabbitMQ)는 미도입. 내부 이벤트는 메서드 호출로 처리.
2. **감사 로깅 의무화**: LLM 호출, 도구 사용, 액션 실행, 정책 평가 모든 주요 이벤트에 AuditLogger가 관여하여 추적성 보장.
3. **확장 대비**: 향후 이벤트 버스 도입 시 현재의 내부 이벤트를 메시지 큐로 전환 가능한 구조. ActionExecuted, WorkflowRunCompleted 등은 이벤트 계약이 이미 정의됨.
