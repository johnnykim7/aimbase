1. Aimbase란?

Aimbase는 AI 기능을 앱에 쉽게 붙일 수 있게 해주는 AI 엔진입니다.

AXOPM이 프로젝트 관리의 "본체"라면, Aimbase는 그 안에서 돌아가는 "AI 두뇌" 역할을 합니다.

쉽게 비유하면 AXOPM = 자동차 본체  /  Aimbase = 자율주행 AI 시스템
자동차는 그대로인데, AI가 추가되면 네비게이션·자동 주행·위험 감지 같은 똑똑한 기능이 생깁니다.

Aimbase가 AXOPM에 더해주는 AI 능력

💬 대화형 AI 어시스턴트

"이번 스프린트 진행률 알려줘", "긴급 이슈 정리해줘" — 자연어로 프로젝트 현황을 물어볼 수 있습니다.

📚 지식 검색 (RAG)

프로젝트 문서, 위키, 회의록을 AI가 이해합니다. "이전에 비슷한 이슈 없었어?" → 의미 기반 검색.

📝 문서 자동 생성

주간 보고서, 회의록 요약, 릴리스 노트를 AI가 자동 작성. Word, PDF, PPT 형식 지원.

🤖 MCP 도구 통합

AXOPM의 35개 MCP 도구를 통해 AI가 이슈 생성, 상태 변경, 보고서 작성까지 직접 실행합니다.

🛡️ 안전 장치 (가드레일)

개인정보 자동 마스킹, 민감 정보 차단, 비용 한도. AI를 안전하게 사용하기 위한 정책 엔진 내장.

🎙️ 음성 → 텍스트

회의 녹음을 텍스트로 변환(STT) → AI가 요약·액션 아이템 추출 → 이슈/태스크 자동 등록.

2. AI가 바꾸는 것들

AXOPM에 Aimbase를 연결하면, 기존에 사람이 직접 하던 작업을 AI가 보조하거나 자동화합니다.

🔴 Before (AI 없이)
  • 목표상 구조(6단계) 수동 설계 — 반나절 소요
  • 이슈 분류/우선순위 라벨링 직접 수행
  • 주간 보고서 매주 수동 작성 (2~3시간)
  • 회의록 직접 타이핑 후 액션 아이템 수동 추출
  • 크리티컬 패스·병목 발견은 이미 지연된 후
  • 리스크 등록은 회의에서 누군가 기억해낼 때만
→
🟢 After (Aimbase 연결)
  • 기획서 올리면 AI가 6단계 구조 자동 생성 — 5분
  • 이슈 등록 시 AI가 자동 분류 + 우선순위 + 담당자 추천
  • 업무일지 기반 주간 보고서 1클릭 생성 — 5분
  • 회의 녹음 → 회의록 + 이슈 N건 자동 생성
  • 실시간 일정 지연 감지 + 사전 경고
  • 일정 지연 감지 시 리스크 자동 등록 + 알림

OPM 파이프라인 뷰
6단계 진행 시각화		 AI 종합 건강도 점수 "프로젝트 건강도: 72/100 (일정 -15, 리스크 -8, 이슈 -5). 주의 필요."
동작 방식 AXOPM 대시보드 API(get_project_dashboard, get_project_health)에서 데이터를 수집 → Aimbase AI가 분석·요약 → 자연어 브리핑으로 표시

AI 기능설명사용자 경험
기획서 → 구조 자동 생성 기획 문서를 업로드하면 AI가 분석하여 6단계 구조를 자동으로 세팅 "이 기획서 기반으로 목표상 세팅해줘" → 목표상/성과/산출물/활동 일괄 생성
빌더 AI 보조 6단계 빌더 위저드에서 각 단계별 AI 추천 성과 정의 시 "AI 추천: 이 목표상에 적합한 성과 3건" 제안
확정 의사결정 보조 확정(Confirm) 시 AI가 관련 데이터 분석 → Proceed/Pivot/Finish/Stop 추천 "AI 분석: 산출물 3건 중 2건 완료, 활동 잔여 1건. Proceed 추천."
검토 초안 생성 검토(Review) 작성 시 AI가 관련 활동·산출물 데이터 기반 초안 생성 "검토 초안 생성" 버튼 → AI가 실적·이슈·리스크 요약 초안
💡 시나리오: "이 기획서 기반으로 목표상 세팅해줘"
  1. AI가 기획 문서를 분석 (요구사항, 마일스톤, 산출물 추출)
  2. create_target_image → 목표상 생성
  3. create_outcome × N → 성과 항목 생성
  4. create_output × N → 산출물 항목 생성
  5. create_activity × N → 활동 항목 생성 (담당자/마감일 추론)
  6. 결과 요약 반환: "목표상 1건, 성과 4건, 산출물 12건, 활동 28건 생성 완료"

5. WBS / 간트차트 — AI 일정 분석

📋 WBS & 간트 AI 분석
WBS 트리 (Phase→WorkPackage→Task), 의존관계(FS/SS/FF/SF), 간트차트, 크리티컬 패스, 베이스라인.
AI 기능설명사용자 경험
크리티컬 패스 분석 의존관계 기반 크리티컬 패스 + 지연 시 전체 영향 계산 "Task-42 지연 시 마일스톤 M3가 5일 밀림" → 간트에 빨간 경고
일정 지연 위험 감지 진행률 대비 남은 기간 분석 → 위험 항목 자동 표시 "AI 경고: 3개 Task가 예정 대비 뒤처짐" 알림
베이스라인 비교 분석 현재 일정 vs 베이스라인 차이를 AI가 요약 "베이스라인 대비 전체 2.3일 지연. 주 원인: 설계 Phase 초과."
WBS 자동 생성 목표상/산출물 기반으로 WBS 트리 자동 구성 빌더 완료 시 → WBS Phase/WorkPackage/Task 자동 연계 생성
💡 시나리오: "일정 지연 위험 있는 태스크 찾아줘"
  1. get_critical_path → 크리티컬 패스 태스크 목록 확인
  2. get_schedule_risks → 지연 위험 항목 필터
  3. AI가 종합 분석: "Task-18(API 개발)이 크리티컬 패스 위에 있고 2일 지연 중. Task-22(테스트)에 영향."

AI 기능설명사용자 경험
자동 분류 + 라벨링 이슈 내용을 읽고 유형(BUG/FEATURE/IMPROVEMENT/QUESTION) 자동 태깅 라벨 드롭다운에 AI 추천값이 미리 선택됨
우선순위 추천 영향도·긴급도·크리티컬 패스 연관 분석 → 우선순위 제안 "AI 추천: Critical (크리티컬 패스 Task와 연관)" 배지
담당자 추천 과거 이슈 해결 이력 + 현재 업무량 기반 적합한 담당자 "AI 추천: 김개발 (유사 이슈 5건 해결, 현재 여유)"
유사 이슈 검색 제목/설명과 비슷한 기존 이슈를 AI가 의미 검색 "혹시 이 이슈와 같은 건가요? — #PROJ-89 (유사도 87%)"
해결 방안 제안 유사 이슈 해결 이력 + 프로젝트 문서에서 관련 정보 검색 "관련 문서: API 설계서 3.2절 / 유사 해결: #PROJ-156"
핵심 포인트 AI는 "추천"만 합니다. 최종 결정은 항상 사람이 합니다. AI 추천을 수락·무시·수정 모두 가능.
💡 시나리오: "오늘 회의록 정리해서 이슈 만들어줘"
  1. 회의록 텍스트 또는 녹음 파일 분석
  2. 액션 아이템 추출 (누가, 무엇을, 언제까지)
  3. create_issue × N건 호출 — 담당자/우선순위/마감일 자동 배정
  4. create_entity_link로 해당 Activity에 연결
  5. "이슈 5건 생성 완료: #PROJ-45(김개발), #PROJ-46(박디자인)..."

8. 리스크 관리 — AI 자동 감지

⚠️ 리스크 등록부 & 매트릭스 AI 자동
리스크 등록·평가 (확률×영향 5×5 매트릭스), 대응 기록, 리스크 규칙 엔진.
AI 기능설명사용자 경험
리스크 자동 등록 AI가 일정 지연, 이슈 급증 등을 감지하면 리스크를 자동 등록 "AI 감지 리스크: WBS Task-28이 3일 연속 진척 없음. 크리티컬 패스 영향."
Top N 리스크 요약 "프로젝트 Top 5 리스크 알려줘" → 스코어 기준 정렬 + 대응 현황 리스크별 현황 + AI 추천 대응 방안
재평가 보조 상황 변화에 따른 리스크 확률/영향 재평가 AI 추천 "API 모듈 이슈 해결됨. 리스크 R-05 확률 High→Low 변경 추천."
💡 시나리오: AI가 일정 지연 감지 → 리스크 자동 등록
  1. get_schedule_risks로 지연 태스크 감지
  2. get_critical_path로 크리티컬 패스 영향 확인
  3. create_risk 자동 호출 — "WBS Task X가 3일 지연, 크리티컬 패스에 영향"
  4. 리스크 매트릭스에 자동 반영 + 담당자 알림

릴리스 노트 완료 이슈/태스크 수집 → 카테고리별 릴리스 노트 자동 생성
💡 시나리오: "이번 주 업무일지 기반으로 주간보고 작성해줘"
  1. get_my_worklogs → 이번 주 업무일지 수집
  2. search_issues → 이번 주 변경된 이슈 수집
  3. AI가 카테고리별 정리 (완료/진행중/이슈/다음주 계획)
  4. generate_weekly_report → 보고서 초안 생성
  5. "주간 보고서 초안 완료. 확인 후 공유하세요."
업무일지는 삭제 불가 (감사 원칙) AXOPM은 업무일지 삭제를 허용하지 않습니다. AI가 생성한 일지도 동일한 원칙이 적용됩니다. 수정은 가능하지만 삭제는 불가.

  • 기획서 → OPM 6단계 자동 생성

  • 주간 보고서 1클릭 생성
  • 회의 녹음 → 회의록 + 이슈 생성
  • 리스크 자동 감지·등록

  • OKR ↔ 산출물 연계 분석
  • 확정 의사결정 AI 보조

16. 자주 묻는 질문 (FAQ)

Q. AI가 이슈를 잘못 분류하면?

AI 분류는 "추천"입니다. 사용자가 언제든 수정할 수 있고, 수정 이력이 쌓이면 AI가 더 정확해집니다.

Q. AI가 만든 이슈나 업무일지를 삭제할 수 있나요?

이슈는 삭제 가능합니다. 업무일지는 AXOPM의 감사 원칙에 따라 삭제 불가합니다 (수정은 가능).

Q. AI가 쓰기(W) 도구로 잘못된 데이터를 만들 수 있지 않나요?

쓰기 도구 실행 전 사용자 확인을 거치도록 설정할 수 있습니다. 또한 Aimbase 정책 엔진에서 특정 작업은 "승인 필요(REQUIRE_APPROVAL)" 정책을 걸 수 있습니다.

Q. 기존 AXOPM 사용법이 바뀌나요?

기존 화면과 워크플로우는 그대로 유지됩니다. AI 기능은 기존 화면 위에 "추가"되는 형태입니다.

Q. 프로젝트 데이터가 외부로 유출되나요?

Aimbase는 개인정보를 자동 마스킹한 후 AI에 전달합니다. 로컬 AI 모델(Ollama)을 사용하면 외부 API 호출 없이 사내에서만 동작합니다.

Q. MCP 도구 35개를 다 써야 하나요?

아닙니다. 필요한 도구만 선택적으로 활성화할 수 있습니다. Phase 1에서는 읽기(R) 도구만 활성화하고, 익숙해지면 쓰기(W) 도구를 추가하는 것을 권장합니다.
