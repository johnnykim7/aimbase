1. LexAI Engine이 제공하는 기능

LexAI Engine은 Lexflow의 모든 AI 기능을 담당하는 백엔드 AI 엔진입니다. Lexflow 앱은 LexAI Engine REST API만 호출하면 됩니다.

🤖 LLM 통합 관리

  • Claude, GPT, Ollama 단일 API
  • 모델 자동 라우팅 / 폴백
  • SSE 스트리밍 응답
  • 구조화 출력 (JSON Schema)
  • 도구 자동 호출 (최대 5회)

📚 RAG 파이프라인

  • 문서 업로드 → 자동 파싱·청킹·임베딩
  • BM25 + 벡터 하이브리드 검색
  • Cross-encoder 재랭킹
  • Contextual Retrieval (검색 정확도 49%↑)
  • Parent-Child 계층 검색
  • 시맨틱 캐시

📝 문서 자동 생성

  • 소장/고소장/계약서 AI 생성
  • DOCX, PDF, PPTX, XLSX 지원
  • 템플릿 관리 (변수 바인딩)
  • 스키마 기반 문서 생성
  • 테마 프리셋

🛡️ 정책 엔진 (가드레일)

  • PII 자동 마스킹 (Presidio)
  • 금지어 차단 (DENY)
  • 승인 필요 (REQUIRE_APPROVAL)
  • 레이트 리밋 (비용 한도)
  • 감사 로깅 (법적 추적성)

⚙️ 워크플로우 엔진

  • DAG 기반 멀티스텝 자동화
  • LLM 호출 / 도구 실행 / 조건 분기
  • 사람 승인 스텝 (HUMAN_INPUT)
  • 병렬 실행
  • 실행 이력 추적

🎙️ 음성 + 평가

  • STT — 상담 녹음 텍스트 변환
  • TTS — 판결문 읽어주기
  • RAGAS 품질 평가
  • LLM Judge (환각/독성 탐지)
  • 프롬프트 A/B 비교
멀티테넌시 내장 Database-per-Tenant 아키텍처로 법률사무소별 데이터 완전 격리. 플랜(Starter/Pro/Enterprise)별 API 쿼터 자동 관리.

2. 연동 방식 3가지

A. REST API 직접 호출

Lexflow BE → LexAI Engine REST API

가장 간단, 권장
HTTP 호출만으로 모든 AI 기능 사용

B. MCP 서버 래핑

Lexflow 전용 MCP 서버 구축
→ LexAI Engine API를 MCP Tool로 노출

AI 에이전트가 도구를 자율 선택할 때

C. LexAI Engine 내부 MCP 직접 연결

Lexflow → Python Sidecar MCP

39개 내부 도구를 세밀하게 제어할 때

권장 전략: A 먼저 → 필요 시 B/C 추가 대부분 REST API로 충분합니다. MCP는 "에이전트 자율 도구 선택"이 필요한 고급 시나리오에서 사용.

10. MCP 서버 코드 예시

lexflow_mcp_server.py
from mcp.server import Server
from mcp.types import Tool, TextContent
import httpx

LEXAI_ENGINE = "http://localhost:8080"
server = Server("lexflow-ai")

class LexAIClient:
    def __init__(self, tenant_id, token):
        self.headers = {
            "Authorization": f"Bearer {token}",
            "X-Tenant-Id": tenant_id,
            "Content-Type": "application/json"
        }

    async def _post(self, path, body):
        async with httpx.AsyncClient(timeout=60) as c:
            r = await c.post(f"{LEXAI_ENGINE}{path}", json=body, headers=self.headers)
            return r.json()

    async def chat(self, messages, **kw):
        return await self._post("/api/v1/chat/completions", {"messages": messages, **kw})

    async def search(self, query, sources, top_k=5):
        return await self._post("/api/v1/knowledge-sources/search",
            {"query": query, "source_ids": sources, "top_k": top_k, "search_type": "HYBRID"})

    async def gen_doc(self, prompt, fmt="docx"):
        return await self._post("/api/v1/documents/generate-json",
            {"prompt": prompt, "format": fmt})

client = LexAIClient("daejung-law", "JWT_TOKEN")

# ──── MCP Tool 등록 ────
@server.tool()
async def analyze_case(case_summary: str, case_type: str) -> list[TextContent]:
    """사건 분석 — 쟁점 추출, 관련 법조문, 위험도 평가"""
    r = await client.chat([
        {"role":"system", "content":"법률 AI. 쟁점·관련법·위험도를 JSON으로 분석."},
        {"role":"user", "content":f"[{case_type}] {case_summary}"}
    ], knowledge_source_ids=["ks-civil-law"],
       response_format={"type":"json_schema", "schema_id":"case-analysis"})
    return [TextContent(type="text", text=str(r["data"]))]

@server.tool()
async def search_precedents(query: str, top_k: int = 5) -> list[TextContent]:
    """유사 판례 검색 — 벡터+키워드 하이브리드"""
    r = await client.search(query, ["ks-precedents"], top_k)
    return [TextContent(type="text", text=str(r["data"]))]

@server.tool()
async def draft_complaint(case_info: str, case_type: str) -> list[TextContent]:
    """소장/고소장 AI 초안 생성 (DOCX)"""
    doc_type = "고소장" if case_type == "형사" else "소장"
    r = await client.gen_doc(f"{case_type} {doc_type} 작성:\n{case_info}")
    return [TextContent(type="text", text=str(r["data"]))]

@server.tool()
async def review_document(document_text: str) -> list[TextContent]:
    """서면 AI 검토 — RAG 법조문 교차 검증"""
    r = await client.chat([
        {"role":"system", "content":"법률 문서 검토 전문가. JSON으로 결과 반환."},
        {"role":"user", "content":f"검토:\n{document_text}"}
    ], knowledge_source_ids=["ks-civil-law"],
       response_format={"type":"json_schema", "schema_id":"doc-review"})
    return [TextContent(type="text", text=str(r["data"]))]

case_info: str, case_type: str → base64 DOCX
review_document — 서면 AI 검토 (RAG 교차 검증)
document_text: str → JSON (score, issues, recommendation)
summarize_case — 사건 현황 요약 브리핑
case_data: str → 자연어 요약
transcribe_audio — 상담 녹음 텍스트 변환 (STT)
audio_base64: str, language: str="ko" → 텍스트
generate_report — 사건 결과 리포트 생성 (PDF)
case_data: str, template_id: str → base64 PDF
ingest_document — 문서를 RAG 지식소스에 추가
source_id: str, file_base64: str → 성공 여부

12. 내부 MCP 도구 — RAG 핵심 (13개)

LexAI Engine Python Sidecar (localhost:8000)에서 직접 호출 가능한 내부 MCP 도구입니다.

chunk_document — 문서 시맨틱 청킹 (semantic/recursive/fixed)
content: str, strategy: str="semantic", config: str="{}"
embed_texts — 텍스트 임베딩 벡터 생성 (KoSimCSE 기본, 768d)
texts: list[str], model: str=""
search_hybrid — BM25 + 벡터 하이브리드 검색 (RRF 융합)
query: str, source_id: str, top_k: int=5, vector_weight: float=0.7, keyword_weight: float=0.3
rerank_results — Cross-encoder 재랭킹
query: str, documents: str (JSON), top_k: int=5
ingest_document — 전체 인제스션 파이프라인 (파싱→청킹→임베딩→저장)
source_id: str, content: str, chunking_strategy: str="semantic", embedding_model: str=""
ingest_file — 파일 기반 인제스션 (pdf/docx/pptx/csv/txt/md/html)
source_id: str, file_path: str, storage_base_path: str
transform_query — 쿼리 변환 (HyDE/Multi-query/Step-back)
query: str, strategy: str="multi_query"
parse_document — 문서 파싱 (base64 → 텍스트+메타데이터)
file_content: str, file_type: str=""
compress_context — 컨텍스트 압축 (유사도 낮은 문장 제거)
query: str, documents: str (JSON), similarity_threshold: float=0.5
self_rag_search — Self-RAG (자동 쿼리 정제 + 품질 평가)
query: str, source_id: str, top_k: int=5, max_iterations: int=2
embed_multimodal — CLIP 멀티모달 임베딩 (텍스트+이미지)
items: str (JSON: [{type, content}])
finetune_embeddings — 도메인 특화 임베딩 파인튜닝
training_data: str (JSON: [{query, positive, negative?}]), epochs: int=3
scrape_url — 웹 스크래핑 (basic/js_render/sitemap)
url: str, mode: str="basic", max_pages: int=10

13. 내부 MCP 도구 — RAG 고급 (6개)

contextual_chunk — Contextual Retrieval 청킹 (LLM 컨텍스트 접두사 부여, 검색 정확도 49%↑)
