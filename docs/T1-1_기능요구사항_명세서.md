## 기능 상세

### 모듈: 채팅 (Chat)

#### PRD-001 | LLM 채팅 완료 (동기)
- **모듈**: 채팅 > 채팅완료
- **설명**: 사용자 메시지를 LLM에 전달하고 동기적으로 완료 응답을 반환한다. 세션 기반 대화 이력 관리, RAG 컨텍스트 주입, 도구 호출 루프를 포함한다.
- **입력**: model, session_id, messages[{role, content}], stream=false, actions_enabled, connection_id, rag_source_id
- **출력**: {id, model, session_id, content[], actions_executed[], usage{input_tokens, output_tokens, cost_usd}}
- **제약조건**: 도구 호출 루프 최대 5회 반복; 컨텍스트 윈도우 토큰 제한 내 트리밍
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-002 | LLM 채팅 스트리밍 (SSE)
- **모듈**: 채팅 > 스트리밍
- **설명**: Server-Sent Events(SSE)를 통해 LLM 응답을 실시간 스트리밍으로 반환한다.
- **입력**: model, session_id, messages[], stream=true, actions_enabled, connection_id
- **출력**: SSE 이벤트 스트림 (text/event-stream)
- **제약조건**: 스트리밍 중 도구 호출 시 중간 결과 포함
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

### 모듈: 연결 (Connections)

#### PRD-003 | 연결 목록 조회
- **모듈**: 연결 > 조회
- **설명**: 테넌트에 등록된 외부 서비스 연결(LLM, DB, 메시징 등) 목록을 페이지네이션으로 조회한다.
- **입력**: page, size, type(선택)
- **출력**: Page<ConnectionEntity>
- **제약조건**: 테넌트 격리 (TenantContext 기반)
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-004 | 연결 생성
- **모듈**: 연결 > 생성
- **설명**: 새로운 외부 서비스 연결을 등록한다. 어댑터 유형(openai, anthropic, ollama, postgresql, slack 등)별 설정을 JSONB로 저장한다.
- **입력**: id, name, adapter, type(llm/write/notify), config(JSONB)
- **출력**: ConnectionEntity
- **제약조건**: adapter + type 조합 유효성 검증; config 필드는 어댑터별 상이
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-005 | 연결 상세 조회
- **모듈**: 연결 > 상세
- **설명**: 특정 연결의 상세 정보를 조회한다.
- **입력**: id (path)
- **출력**: ConnectionEntity
- **제약조건**: 테넌트 내 존재하는 ID만 조회 가능
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

#### PRD-006 | 연결 수정
- **모듈**: 연결 > 수정
- **설명**: 기존 연결의 이름, 설정 등을 변경한다.
- **입력**: id (path), ConnectionRequest
- **출력**: ConnectionEntity (갱신됨)
- **제약조건**: 없음
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

#### PRD-007 | 연결 삭제
- **모듈**: 연결 > 삭제
- **설명**: 등록된 연결을 삭제한다.
- **입력**: id (path)
- **출력**: 204 No Content
- **제약조건**: 워크플로우나 라우팅에서 참조 중인 연결은 삭제 불가 (향후)
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

#### PRD-008 | 연결 헬스체크
- **모듈**: 연결 > 헬스체크
- **설명**: 특정 연결의 연결 상태를 테스트하고 응답 시간을 측정한다.
- **입력**: id (path)
- **출력**: {ok: boolean, latencyMs: number}
- **제약조건**: 타임아웃 설정 적용
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

### 모듈: MCP (Model Context Protocol)

- **모듈**: MCP > 조회
- **설명**: 등록된 MCP 서버 목록을 조회한다.

- **출력**: Page<MCPServerEntity>
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-010 | MCP 서버 등록
- **모듈**: MCP > 등록

- **출력**: MCPServerEntity

- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-013 | MCP 도구 탐색

- **설명**: MCP 서버에 연결하여 사용 가능한 도구 목록을 자동 탐색하고 ToolRegistry에 등록한다.
- **입력**: id (path)
- **출력**: {serverId, toolCount, tools[{name, description, inputSchema}]}
- **제약조건**: 서버 연결 실패 시 에러 반환
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

### 모듈: 정책 (Policies)

#### PRD-020 | 정책 목록 조회
- **모듈**: 정책 > 조회
- **설명**: 정책 목록을 우선순위 기반으로 조회한다.
- **입력**: page, size, domain(선택)
- **출력**: Page<PolicyEntity>
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-021 | 정책 생성
- **모듈**: 정책 > 생성
- **설명**: 새로운 정책을 생성한다. 정책 규칙(DENY, REQUIRE_APPROVAL, RATE_LIMIT, TRANSFORM, LOG, ALLOW)과 SpEL 조건을 정의한다.
- **입력**: id, name, domain, priority, matchRules{intents[], adapters[]}, rules[{type, condition, message, config}]
- **출력**: PolicyEntity
- **제약조건**: priority 0-100; condition은 SpEL 표현식; matchRules 빈 경우 전체 매칭
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-025 | 정책 활성화/비활성화
- **모듈**: 정책 > 토글
- **설명**: 정책의 활성 상태를 변경한다.

- **모듈**: 액션 > 정책제어
- **설명**: 액션 실행 전 PolicyEngine으로 정책을 평가한다. DENY 시 거부, REQUIRE_APPROVAL 시 승인 대기, TRANSFORM 시 PII 마스킹 적용. **v2.0: PII 탐지/마스킹을 Python MCP Server(Safety)의 Presidio로 위임한다.** 정책 엔진 코어와 RateLimiter는 Spring에 잔류.
- **입력**: ActionRequest, PolicyEngine
- **출력**: PolicyResult{allowed, action, transforms}
- **제약조건**: PII 마스킹은 Python Presidio로 위임 (MCP 직접 호출); RateLimiter는 Redis 기반 슬라이딩 윈도우
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료 (v2.0 PII 이관 예정)
- **관련 CR**: CR-002

### 모듈: Python-RAG (MCP Server 1: RAG Pipeline)

#### PY-001 | 시맨틱 청킹
- **모듈**: Python-RAG > 청킹
- **설명**: 문서를 의미 단위로 분할한다. 문장 경계를 인식하고, 임베딩 유사도 기반으로 의미적으로 연관된 문장을 하나의 청크로 묶는다.
- **입력**: {content: string, strategy: "semantic"|"recursive"|"fixed", config: {max_chunk_size, similarity_threshold}}
- **출력**: {chunks[{content, metadata, token_count}]}
- **제약조건**: LlamaIndex SemanticSplitter 사용; 한국어 문장 분리 지원
- **우선순위**: High | **MVP**: Y | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-002 | 로컬 임베딩
- **모듈**: Python-RAG > 임베딩
- **설명**: sentence-transformers를 사용하여 외부 API 의존 없이 로컬에서 임베딩을 생성한다. 한국어 특화 모델(KoSimCSE 등) 지원.
- **입력**: {texts: string[], model: string}
- **출력**: {embeddings: float[][], model: string, dimensions: number}
- **제약조건**: 기본 모델 KoSimCSE; OpenAI 임베딩 API도 병행 지원; GPU 가속 선택적
- **우선순위**: High | **MVP**: Y | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-003 | 리랭킹
- **모듈**: Python-RAG > 리랭킹
- **설명**: 초기 검색 결과를 cross-encoder 모델로 재정렬하여 정확도를 향상시킨다.
- **입력**: {query: string, documents[{content, metadata}], top_k: number, model: string}
- **출력**: {results[{content, metadata, rerank_score}]}
- **제약조건**: cross-encoder/ms-marco-MiniLM-L-6-v2 기본; Cohere Rerank API도 지원
- **우선순위**: High | **MVP**: Y | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-004 | 하이브리드 검색
- **모듈**: Python-RAG > 하이브리드검색
- **설명**: BM25 키워드 검색과 벡터 시맨틱 검색을 결합하여 검색 품질을 향상시킨다. RRF(Reciprocal Rank Fusion)로 점수를 통합한다.
- **입력**: {query: string, source_id: string, top_k: number, vector_weight: float, keyword_weight: float}
- **출력**: {results[{content, metadata, vector_score, keyword_score, combined_score}]}
- **제약조건**: rank_bm25 라이브러리 사용; pgvector 코사인 유사도와 결합
- **우선순위**: High | **MVP**: Y | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-005 | 쿼리 변환
- **모듈**: Python-RAG > 쿼리변환
- **설명**: 사용자 질문을 검색에 최적화된 쿼리로 변환한다. HyDE(Hypothetical Document Embedding), Multi-Query 전략 지원.
- **입력**: {query: string, strategy: "hyde"|"multi_query"|"step_back", llm_config: {model, connection_id}}
- **출력**: {original_query, transformed_queries: string[], strategy_used}
- **제약조건**: LLM 호출 필요 (Spring 오케스트레이터 경유); 추가 토큰 비용 발생
- **우선순위**: Medium | **MVP**: N | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-012 | 파인튜닝 파이프라인
- **모듈**: Python-RAG > 파인튜닝
- **설명**: 도메인 특화 임베딩 모델을 학습한다. 기존 검색 로그를 활용하여 양성/음성 쌍을 생성하고 sentence-transformers로 파인튜닝한다.
- **입력**: {base_model: string, training_data[{query, positive, negative}], epochs, batch_size}
- **출력**: {model_path: string, metrics: {loss, accuracy}}
- **제약조건**: GPU 필요; 학습 완료 후 PY-002 모델로 교체 가능
- **우선순위**: Low | **MVP**: N | **상태**: 미구현
- **관련 CR**: CR-002

### 모듈: Python-평가 (MCP Server 2: Evaluation)

#### PY-006 | RAG 품질 평가
- **모듈**: Python-평가 > RAG평가
- **설명**: RAG 응답의 품질을 자동 측정한다. Faithfulness(근거 충실도), Answer Relevancy(응답 적절성), Context Precision(컨텍스트 정확도), Context Recall(컨텍스트 재현율)을 평가한다.
- **입력**: {question: string, answer: string, contexts: string[], ground_truth: string(선택)}
- **출력**: {faithfulness: float, relevancy: float, context_precision: float, context_recall: float}
- **제약조건**: RAGAS 라이브러리 사용; 평가 자체에 LLM 호출 필요 (비용 발생)
- **우선순위**: High | **MVP**: Y | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-007 | LLM 출력 평가
- **모듈**: Python-평가 > LLM평가
- **설명**: LLM 응답의 환각(hallucination), 유해성(toxicity), 편향성(bias)을 자동 탐지한다.
- **입력**: {input: string, output: string, context: string(선택), metrics: string[]}
- **출력**: {hallucination_score: float, toxicity_score: float, bias_score: float, details: {}}
- **제약조건**: DeepEval 라이브러리 사용; 메트릭 선택적 적용 가능
- **우선순위**: High | **MVP**: Y | **상태**: 미구현
- **관련 CR**: CR-002

#### PY-008 | 프롬프트 회귀 테스트
- **모듈**: Python-평가 > 회귀테스트
- **설명**: 프롬프트 변경 시 품질 변화를 A/B 비교한다. 테스트 셋에 대해 기존 프롬프트와 새 프롬프트의 응답 품질을 비교 평가한다.
- **입력**: {test_cases[{input, expected_output}], prompt_a: {id, version}, prompt_b: {id, version}, metrics: string[]}
- **출력**: {summary: {prompt_a_avg, prompt_b_avg, improvement_pct}, details[{input, score_a, score_b}]}
- **제약조건**: 기존 프롬프트 관리(PRD-027~032)와 연동; 테스트 셋 관리 필요
- **우선순위**: Medium | **MVP**: N | **상태**: 미구현
- **관련 CR**: CR-002
