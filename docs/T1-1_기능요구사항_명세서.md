## 기능 상세

### 모듈: 채팅 (Chat)

#### PRD-001 | LLM 채팅 완료 (동기)
- **모듈**: 채팅 > 채팅완료
- **설명**: 사용자 메시지를 LLM에 전달하고 동기적으로 완료 응답을 반환한다. 세션 기반 대화 이력 관리, RAG 컨텍스트 주입, 도구 호출 루프를 포함한다. **v2.5: response_format 파라미터로 구조화된 JSON 응답을 요청할 수 있다.**
- **입력**: model, session_id, messages[{role, content}], stream=false, actions_enabled, connection_id, rag_source_id, **response_format(선택)**
- **출력**: {id, model, session_id, content[**{type: "text"|"structured", ...}**], actions_executed[], usage{input_tokens, output_tokens, cost_usd}}
- **제약조건**: 도구 호출 루프 최대 5회 반복; 컨텍스트 윈도우 토큰 제한 내 트리밍; **response_format 지정 시 스키마 검증 후 반환**
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-007

#### PRD-002 | LLM 채팅 스트리밍 (SSE)
- **모듈**: 채팅 > 스트리밍
- **설명**: Server-Sent Events(SSE)를 통해 LLM 응답을 실시간 스트리밍으로 반환한다.
- **입력**: model, session_id, messages[], stream=true, actions_enabled, connection_id
- **출력**: SSE 이벤트 스트림 (text/event-stream)
- **제약조건**: 스트리밍 중 도구 호출 시 중간 결과 포함
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

### 모듈: 연결 (Connections)

#### PRD-003 | 연결 목록 조회
- **모듈**: 연결 > 조회
- **설명**: 테넌트에 등록된 외부 서비스 연결(LLM, DB, 메시징 등) 목록을 페이지네이션으로 조회한다.
- **입력**: page, size, type(선택)
- **출력**: Page<ConnectionEntity>
- **제약조건**: 테넌트 격리 (TenantContext 기반)
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-004 | 연결 생성
- **모듈**: 연결 > 생성
- **설명**: 새로운 외부 서비스 연결을 등록한다. 어댑터 유형(openai, anthropic, ollama, postgresql, slack 등)별 설정을 JSONB로 저장한다.
- **입력**: id, name, adapter, type(llm/write/notify), config(JSONB)
- **출력**: ConnectionEntity
- **제약조건**: adapter + type 조합 유효성 검증; config 필드는 어댑터별 상이
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료

#### PRD-005 | 연결 상세 조회
- **모듈**: 연결 > 상세
- **설명**: 특정 연결의 상세 정보를 조회한다.
- **입력**: id (path)
- **출력**: ConnectionEntity
- **제약조건**: 테넌트 내 존재하는 ID만 조회 가능
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

#### PRD-006 | 연결 수정
- **모듈**: 연결 > 수정
- **설명**: 기존 연결의 이름, 설정 등을 변경한다.
- **입력**: id (path), ConnectionRequest
- **출력**: ConnectionEntity (갱신됨)
- **제약조건**: 없음
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

#### PRD-007 | 연결 삭제
- **모듈**: 연결 > 삭제
- **설명**: 등록된 연결을 삭제한다.
- **입력**: id (path)
- **출력**: 204 No Content
- **제약조건**: 워크플로우나 라우팅에서 참조 중인 연결은 삭제 불가 (향후)
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

#### PRD-008 | 연결 헬스체크
- **모듈**: 연결 > 헬스체크
- **설명**: 특정 연결의 연결 상태를 테스트하고 응답 시간을 측정한다.
- **입력**: id (path)
- **출력**: {ok: boolean, latencyMs: number}
- **제약조건**: 타임아웃 설정 적용
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료

- **모듈**: 라우팅 > IntentClassifier

- **입력**: String userMessage, List<UnifiedMessage> context
- **출력**: IntentComplexity enum (SIMPLE, MODERATE, COMPLEX)
- **제약조건**: 1차는 규칙 기반(LLM 호출 없음); 향후 LLM 기반 2차 분류 확장점 제공
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-012

#### PRD-125 | Smart Model Routing
- **모듈**: 라우팅 > ModelRouter
- **설명**: model="auto"일 때 IntentClassifier 결과에 따라 최적 모델을 자동 선택한다. SIMPLE→Haiku($0.25/1M), MODERATE→Sonnet($3/1M), COMPLEX→Opus($15/1M). routing_config DB의 rules JSONB에 복잡도→모델 매핑을 저장하여 테넌트별 커스터마이징 가능.
- **입력**: LLMRequest (model="auto"), routing_config
- **출력**: LLMAdapter (복잡도에 맞는 모델의 어댑터)
- **제약조건**: routing_config가 없으면 기존 동작(기본 모델) 유지; 비용 예산 초과 시 한 단계 낮은 모델로 다운그레이드
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-012

#### PRD-126 | 대화 요약 생성
- **모듈**: 세션 > ConversationSummarizer
- **설명**: 이전 대화 메시지를 저가 모델(Haiku)로 요약하여 컨텍스트를 압축한다. 시스템 프롬프트로 "핵심 결정사항, 사용자 요청, 맥락을 보존하여 간결하게 요약"을 지시. max_tokens=500.
- **입력**: List<UnifiedMessage> messagesToSummarize, String model
- **출력**: String summaryText
- **제약조건**: 요약 실패 시 기존 동작(단순 삭제)으로 폴백; SYSTEM 메시지는 요약 대상에서 제외
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-012

#### PRD-127 | 요약 주입 및 ContextWindow 확장
- **모듈**: 세션 > ContextWindowManager
- **설명**: 컨텍스트 윈도우의 70% 토큰에 도달하면 ConversationSummarizer를 트리거하여 오래된 메시지를 요약으로 교체한다. 요약본은 SYSTEM 메시지 바로 뒤에 "[이전 대화 요약]" 형태로 주입. 요약 텍스트는 conversation_sessions.summary_text에 저장.
- **입력**: List<UnifiedMessage> messages, int maxTokens
- **출력**: List<UnifiedMessage> (요약 적용된 메시지 목록)
- **제약조건**: 요약 트리거 임계값: 전체 토큰의 70%; 요약 후 목표: 전체 토큰의 50% 이하
- **우선순위**: High | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-012

#### PRD-128 | Exact Match 응답 캐시
- **모듈**: 캐시 > ResponseCacheService
- **설명**: 동일한 질문에 대해 LLM을 다시 호출하지 않도록 응답을 캐시한다. SHA-256(model + systemPromptHash + lastUserMessage) 해시를 키로 Redis에 저장. TTL 1시간 기본, 용도별 조절 가능.
- **입력**: model, systemPrompt, userMessage
- **출력**: Optional<CachedResponse> (캐시 히트 시), void (캐시 저장 시)
- **제약조건**: 대화형 세션(session_id 있는 경우)은 캐시 비활성; 히트 시 ChatResponse에 cacheHit=true 표시
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-012

#### PRD-129 | Semantic Match 응답 캐시
- **모듈**: 캐시 > ResponseCacheService
- **설명**: 의미적으로 유사한 질문에 대해 기존 캐시를 재사용한다. 사용자 메시지를 임베딩하여 pgvector에서 코사인 유사도≥0.95인 캐시 항목을 검색. Exact match 미히트 시 Semantic match 시도.
- **입력**: String userMessage, double threshold (기본 0.95)
- **출력**: Optional<CachedResponse>
- **제약조건**: 임계값 초기 0.98로 보수적 시작, 모니터링 후 조정; response_cache 테이블에 query_embedding vector(1536) 저장
- **우선순위**: Low | **MVP**: N | **상태**: 구현완료
- **관련 CR**: CR-012

#### PRD-130 | 메모리 계층 분리
- **모듈**: 메모리 > MemoryService
- **설명**: 대화 메모리를 4계층으로 분리하여 구조적으로 관리한다. SYSTEM_RULES(항상 주입, 10%), LONG_TERM(세션 간 유지, 15%), SHORT_TERM(현재 대화, 75%), USER_PROFILE(사용자 선호/이력). 각 계층별 토큰 할당량을 두어 경합을 방지한다.
- **입력**: sessionId, userId, MemoryLayer, content
- **출력**: List<MemoryEntry>, List<UnifiedMessage> buildMemoryContext()
- **제약조건**: LONG_TERM은 relevance score 기반 top-K만 주입; conversation_memories 테이블에 memory_type, embedding 저장
- **우선순위**: Medium | **MVP**: Y | **상태**: 구현완료
- **관련 CR**: CR-012
