## Sprint 2: 연결 관리

#### UAT-S2-001: 연결 CRUD 전체 흐름


## Sprint 6: 워크플로우 실행 + 승인

#### UAT-S6-001: 워크플로우 실행 → 승인 → 완료

```bash
# 1. 워크플로우 생성 (HUMAN_INPUT 포함)
curl -X POST http://localhost:8080/api/v1/workflows \
  -H "Content-Type: application/json" \
  -d '{
    "id": "test-wf",
    "name": "Approval Test",
    "steps": [
      {"id": "step1", "type": "LLM_CALL", "config": {"model": "gpt-4o", "prompt": "Say hello"}, "dependsOn": []},
      {"id": "step2", "type": "HUMAN_INPUT", "config": {"approvers": ["admin"]}, "dependsOn": ["step1"]}
    ],
    "errorHandling": {"retryMaxAttempts": 3, "retryDelayMs": 1000}
  }'

# 2. 실행
curl -X POST http://localhost:8080/api/v1/workflows/test-wf/run

# 3. 승인 대기 확인
curl http://localhost:8080/api/v1/admin/approvals

# 4. 승인 처리
curl -X POST http://localhost:8080/api/v1/admin/approvals/{approvalId}/approve
```

**예상 결과**: step1 실행 → step2에서 pending_approval → 승인 후 completed
