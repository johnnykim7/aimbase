## 작성 원칙

1. **사용자 여정** 기반 — 기능 단위가 아닌 업무 흐름 단위
2. **화면 조작 단계**로 기술 — curl/API 등 개발자 요소 배제
3. 정상 흐름 / 에러 흐름 / 비즈니스 규칙 검증 포함
4. Sprint별 5~10개 시나리오 (MVP/High 기능 중심)
5. 시나리오 ID 형식: `UAT-S{Sprint}-{순번}`
