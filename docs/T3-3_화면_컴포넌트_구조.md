## 컴포넌트 구조 + API 매핑

### Dashboard (`/`)

```
AppShell
└── Dashboard
    ├── StatCard × 4 → GET /admin/dashboard (30s interval)
    │   ├── 오늘 토큰
    │   ├── 활성 연결
    │   ├── 승인 대기
    │   └── 오늘 비용
    ├── ActionLogList → GET /admin/action-logs (5s interval)

```

## 공통 컴포넌트

### Layout
- **AppShell**: 전체 레이아웃 (Sidebar + Main Content)
- **Sidebar**: 고정 220px, 네비게이션 (10개 메인 + 3개 Super Admin)
- **PageHeader**: 제목 + 부제 + 액션 버튼

### UI
- **DataTable**: 제네릭 테이블 (columns, data, keyExtractor, loading)
- **Modal**: 오버레이 모달 (open, onClose, title, width)
- **FormField**: 레이블 + 입력 + 힌트
- **ActionButton**: 다형 버튼 (primary/danger/success/ghost, icon, small)
- **StatCard**: 통계 카드 (label, value, sub, color)
- **Badge**: 상태 뱃지 (accent/success/warning/danger/purple/muted, pulse)
- **LoadingSpinner**: 로딩 인디케이터
- **EmptyState**: 빈 상태 안내

## 작성 가이드

1. **트리 형식**: 컴포넌트 계층을 들여쓰기 트리로 표현
2. **API 매핑**: 각 컴포넌트 옆에 `→ HTTP메서드 /경로` 로 API 연결 표시
3. **Props/State 미기재**: 상세 Props는 코드에서 자동 도출
4. **공통 컴포넌트 참조**: 재사용 컴포넌트는 공통 컴포넌트 섹션 참조
