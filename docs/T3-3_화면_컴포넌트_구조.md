## 컴포넌트 구조 + API 매핑

### Dashboard (`/`)

```
AppShell
└── Dashboard
    ├── StatCard × 4 → GET /admin/dashboard (30s interval)
    │   ├── 오늘 토큰
    │   ├── 활성 연결
    │   ├── 승인 대기
    │   └── 오늘 비용
    ├── ActionLogList → GET /admin/action-logs (5s interval)
    │   └── Badge (status)
    ├── ModelUsageChart → GET /admin/usage
    │   └── 모델별 비용 바 차트
    └── ApprovalList → GET /admin/approvals (10s interval)
        ├── ActionButton (승인) → POST /admin/approvals/{id}/approve
        └── ActionButton (거부) → POST /admin/approvals/{id}/reject
```

```
AppShell
└── WorkflowStudio

    │   └── 스텝 유형 6개 (LLM/Tool/Condition/Parallel/Approval/Action) 드래그 소스

    │   └── 실행 시각화 (노드별 상태 색상/아이콘) [FE-005]

    │   ├── 노드 설정 (스텝 유형별 config 폼)
    │   │   ├── LLM 노드: 모델/프롬프트 선택, **응답 포맷 (없음/스키마 선택/인라인 정의)** [CR-007]
    │   │   │   └── **runtime 선택 드롭다운** (NATIVE/SIDECAR/EXTERNAL) [v4.0, CR-029]
    │   │   ├── Tool 노드: 도구 선택
    │   │   │   ├── **네이티브 Tool 1급 노출** (built-in 도구 카드 직접 선택) [v4.0, CR-029]
    │   │   │   ├── **SchemaForm 동적 렌더** (선택된 도구의 input_schema → 동적 폼) [v4.0, CR-029]
    │   │   │   ├── **approval 뱃지** (require_approval=true 도구 경고 표시) [v4.0, CR-029]
    │   │   │   └── **capability filter** (도구 capability 태그별 필터링) [v4.0, CR-029]
    │   │   ├── Condition 노드: 분기 조건식
    │   │   └── Approval 노드: 승인자 설정
    │   └── 워크플로우 속성 패널
    │       ├── 이름, 도메인, 트리거 설정
    │       └── **출력 스키마 (output_schema)** [CR-007, PRD-101]
    │           ├── 드롭다운: 등록된 스키마 선택 → GET /schemas
    │           ├── 인라인 JSON Schema 에디터 (코드 에디터)
    │           └── 스키마 프리뷰 (예상 출력 구조 트리)
    ├── 툴바
    │   ├── ActionButton (저장) → POST/PUT /workflows [FE-004]
    │   ├── ActionButton (실행) → POST /workflows/{id}/run
    │   └── ActionButton (자동 레이아웃) → dagre 레이아웃 적용
    └── flowToWorkflow / workflowToFlow 유틸 [FE-004]
```

## 공통 컴포넌트

### Layout
- **AppShell**: 전체 레이아웃 (Sidebar + Main Content)
- **Sidebar**: 고정 220px, 네비게이션 (18개 메인 + 7개 Super Admin)
- **PageHeader**: 제목 + 부제 + 액션 버튼

### UI
- **DataTable**: 제네릭 테이블 (columns, data, keyExtractor, loading)
- **Modal**: 오버레이 모달 (open, onClose, title, width)
- **FormField**: 레이블 + 입력 + 힌트
- **ActionButton**: 다형 버튼 (primary/danger/success/ghost, icon, small)
- **StatCard**: 통계 카드 (label, value, sub, color)
- **Badge**: 상태 뱃지 (accent/success/warning/danger/purple/muted, pulse)
- **LoadingSpinner**: 로딩 인디케이터
- **EmptyState**: 빈 상태 안내

## 작성 가이드

1. **트리 형식**: 컴포넌트 계층을 들여쓰기 트리로 표현
2. **API 매핑**: 각 컴포넌트 옆에 `→ HTTP메서드 /경로` 로 API 연결 표시
3. **Props/State 미기재**: 상세 Props는 코드에서 자동 도출
4. **공통 컴포넌트 참조**: 재사용 컴포넌트는 공통 컴포넌트 섹션 참조
