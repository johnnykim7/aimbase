## 3. 환경변수 설정

```bash
cd backend
cp .env.example .env
```

**필수 환경변수** (.env 파일 수정):

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `SERVER_PORT` | 8080 | BE 서버 포트 |
| `MASTER_DB_URL` | jdbc:postgresql://localhost:5432/llm_platform_master | Master DB URL |
| `MASTER_DB_USERNAME` | platform | Master DB 사용자 |
| `MASTER_DB_PASSWORD` | platform | Master DB 비밀번호 |
| `TENANT_DB_HOST` | localhost | Tenant DB 호스트 |
| `TENANT_DB_PORT` | 5432 | Tenant DB 포트 |
| `TENANT_DB_USERNAME` | platform | Tenant DB 사용자 |
| `TENANT_DB_PASSWORD` | platform | Tenant DB 비밀번호 |
| `SPRING_REDIS_HOST` | localhost | Redis 호스트 |
| `SPRING_REDIS_PORT` | 6379 | Redis 포트 |
| `ANTHROPIC_API_KEY` | sk-ant-xxx | Anthropic API 키 (**필수**) |
| `OPENAI_API_KEY` | sk-xxx | OpenAI API 키 (**필수**) |
| `JWT_SECRET` | change-me-in-production | JWT 시크릿 |
| `ENCRYPTION_KEY` | change-me-in-production | 암호화 키 |

## 4. 인프라 기동

```bash
cd backend
docker compose up -d
```

**기대 결과**:
```
✔ Container postgres-master   Running
✔ Container postgres-tenant    Running
✔ Container redis              Running
```

**확인**:
```bash
docker compose ps
# 모든 서비스 "running" 확인

# PostgreSQL 접속 확인
docker exec -it postgres-master psql -U platform -d llm_platform_master -c "SELECT 1;"

# Redis 접속 확인
docker exec -it redis redis-cli ping
# → PONG
```

## 8. 자주 발생하는 문제

### PostgreSQL 포트 충돌
```
Error: Bind for 0.0.0.0:5432 failed: port is already allocated
```
**해결**: 기존 PostgreSQL 중지 또는 docker-compose.yml 포트 변경

### Flyway 마이그레이션 실패
```
FlywayException: Validate failed
```
**해결**: `docker compose down -v` 후 볼륨 초기화 재실행

### Redis 연결 실패
```
RedisConnectionException: Unable to connect
```
**해결**: `docker compose up redis -d` 확인, 포트 6379 확인

### OpenAI/Anthropic API 키 오류
```
401 Unauthorized
```
**해결**: .env 파일의 API 키 확인, 잔액/쿼터 확인

## 9. 프로덕션 배포

```bash
# BE 빌드
cd backend
./gradlew :platform-core:bootJar

# Docker 이미지 빌드
docker build -t aimbase:latest -f platform-core/Dockerfile .

# Docker Compose로 전체 기동
docker compose -f docker-compose.yml up -d
```

**프로덕션 필수 변경사항**:
- `JWT_SECRET` → 안전한 랜덤 값
- `ENCRYPTION_KEY` → 안전한 랜덤 값
- DB 비밀번호 → 강력한 비밀번호
- `SPRING_PROFILES_ACTIVE=prod`
