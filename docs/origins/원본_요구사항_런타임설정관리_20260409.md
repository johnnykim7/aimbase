# CR-040 원본 요구사항 — 런타임 설정 관리 (DB + 관리자 UI)

## 발생 배경
FlowGuard 시나리오 추출 3자 벤치마크(Aimbase vs Claude CLI vs OpenClaude)에서
`MAX_ITERATIONS`, `default-max-tokens` 등의 하드코딩 값이 응답 품질에 직접적 영향을 미침을 확인.
현재 이런 값들이 코드 상수 또는 application.yml에 흩어져 있어 변경 시 빌드/재기동 필요.

## 요구사항
서버 재기동 없이 관리자가 런타임에 미세조정할 수 있도록 DB 테이블 + FE 관리 UI 제공.

## 대상 설정값 (현재 파악분 ~15개)

### 오케스트레이터
| 설정 키 | 현재 값 | 현재 위치 | 영향 |
|---------|--------|----------|------|
| `max-tool-iterations` | 30 | application.yml | 도구 루프 최대 횟수 — 복잡 태스크 품질 |
| `default-max-tokens` | 16000 | application.yml | Anthropic 기본 응답 최대 토큰 |
| `tool-result-budget-bytes` | 50KB | ToolCallHandler 하드코딩 | 도구 결과 축약 기준 |
| `tool-result-compaction-threshold` | 80KB | ToolCallHandler 하드코딩 | 축약 트리거 임계값 |

### 세션
| 설정 키 | 현재 값 | 현재 위치 | 영향 |
|---------|--------|----------|------|
| `session-ttl-hours` | 24 | 하드코딩 (BIZ-002) | 세션 유지 시간 |
| `max-messages-per-session` | 500 | 하드코딩 (BIZ-057) | 세션당 메시지 수 제한 |
| `message-body-max-bytes` | 32KB | 하드코딩 (BIZ-057) | 메시지 본문 크기 제한 |

### 컨텍스트 압축
| 설정 키 | 현재 값 | 현재 위치 | 영향 |
|---------|--------|----------|------|
| `snip-percent` | 70 | application.yml | SNIP 압축 시작 |
| `micro-compact-percent` | 85 | application.yml | MICRO_COMPACT 시작 |
| `session-memory-percent` | 91 | application.yml | 세션 메모리 추출 |
| `auto-compact-percent` | 93 | application.yml | AUTO_COMPACT 시작 |
| `blocking-limit-percent` | 98 | application.yml | 블로킹 제한 |

### LLM
| 설정 키 | 현재 값 | 현재 위치 | 영향 |
|---------|--------|----------|------|
| `extended-thinking-budget` | Connection config | DB (connections) | Thinking 토큰 예산 |
| `thinking-mode` | Connection config | DB (connections) | DISABLED/ENABLED/ADAPTIVE |

## 기대 산출물
1. **BE**: `platform_settings` 테이블 (Master DB) + `PlatformSettingsService` (캐시 + 변경 감지)
2. **FE**: 관리자 설정 페이지 (카테고리별 그룹핑, 값 변경 + 즉시 반영)
3. **API**: `GET/PUT /api/v1/platform/settings` (관리자 전용)
4. 설정 변경 시 감사 로그 기록
