## 배경

Aimbase의 BashTool, FileWriteTool, SuggestBackgroundPR, WebSearchTool은 현재 ClaudeCodeTool을 경유해야만 사용 가능하다. 이는:
1. ClaudeCodeTool 프로세스 오버헤드 (subprocess 기동 + CLI 파싱)
2. 네트워크/인증 의존성 (Anthropic API Key 필수)
3. 도구 실행 투명성 부재 (lineage 추적 불가)
4. 정책 엔진 우회 (ClaudeCodeTool 내부 실행은 PolicyEngine 미적용)

## 요구사항

### PRD-241: BashTool
- 워크스페이스 내 셸 명령 직접 실행
- 위험 명령 차단 (rm -rf /, mkfs, shutdown, reboot, dd, format 등)
- 타임아웃 120초, stdout/stderr 분리 캡처
- WorkspaceResolver로 작업 디렉토리 제한
- PermissionLevel.FULL, destructive=true

### PRD-242: FileWriteTool
- 신규 파일 생성 (SafeEditTool은 기존 파일 편집만 지원)
- 기존 파일 존재 시 덮어쓰기 경고 + overwrite 파라미터
- WorkspacePolicyEngine으로 경로 검증
- 부모 디렉토리 자동 생성
- PermissionLevel.RESTRICTED_WRITE

### PRD-243: WebSearchTool
- 에이전트 자율 웹 검색
- Tavily API 우선 (Connection에서 API Key 조회)
- 폴백: DuckDuckGo HTML 파싱 (API Key 불필요)
- 결과: title, url, snippet 구조화 반환
- 최대 10개 결과

### PRD-244: SuggestBackgroundPR
- Git 커밋 + GitHub PR 자동 생성
- ProcessBuilder로 git 명령 실행
- GitHub API (REST) 로 PR 생성
- 토큰은 Connection 테이블에서 조회 (type=GITHUB)
- 브랜치 자동 생성, PR 제목/본문 구조화

### FE-018: 도구 상태 확인 UI
- 기존 도구 목록 페이지에 새 도구 4개 자동 노출 (ToolRegistry 기반)
- 별도 페이지 불필요
