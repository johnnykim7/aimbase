# 원본 요구사항: 문서 생성 사이드카 MCP 도구

> **요청일**: 2026-03-25
> **요청자**: sykim
> **CR 번호**: CR-018

## 배경

Genspark 같은 AI 슬라이드 생성 도구가 LLM으로 python-pptx 코드를 실시간 생성하여 PPTX를 만드는 것을 참고했다. Aimbase에서도 RAG로 읽어들인 지식을 **검색만 하는 것이 아니라 문서로 재생산**하는 기능이 필요하다.

기존 CR-013(보류)에서 "문서 분석/생성 파이프라인"을 Level 1~3으로 정의했으나, 구현 접근법이 확정되지 않아 보류 상태였다. 이번 CR-018에서는 "LLM이 Python 코드를 직접 생성하고 서버에서 실행"하는 접근법으로 구체화한다.

## 핵심 아이디어

- LLM이 문서 생성 라이브러리(python-pptx, python-docx, reportlab 등)를 활용하는 **Python 코드를 직접 생성**
- Python 사이드카가 해당 코드를 **샌드박스 환경에서 실행**하여 파일 생성
- 생성된 파일을 **다운로드 URL로 반환**

이 접근의 장점:
- 템플릿 엔진이나 변환기를 별도로 개발할 필요 없음
- LLM이 요청 맥락에 맞는 코드를 매번 새로 생성하므로 유연성이 높음
- 새로운 포맷 지원 시 라이브러리만 추가하면 됨

## 지원 포맷

| 포맷 | 라이브러리 | 용도 |
|------|-----------|------|
| PPTX | python-pptx | 프레젠테이션, IR자료, 보고서 슬라이드 |
| DOCX | python-docx | 보고서, 제안서, 계약서 |
| PDF | reportlab / weasyprint | 인쇄용 보고서, 인보이스 |
| XLSX | openpyxl | 데이터 분석 결과, 재무 보고서 |
| CSV | csv (표준 라이브러리) | 데이터 내보내기 |
| HTML | Jinja2 | 웹 보고서, 이메일 템플릿 |

## 사용자 시나리오

### 시나리오 1: IR자료 생성
1. 사용자: "이 데이터로 IR자료 만들어줘"
2. LLM: 도구 호출 결정 → `document_generate` MCP 도구 호출
3. LLM: PPTX 생성 python-pptx 코드 작성 (회사 소개, 시장 분석, 재무 지표 슬라이드 등)
4. 사이드카: 코드 실행 → PPTX 파일 생성
5. FE: 다운로드 링크 표시 → 사용자 다운로드

### 시나리오 2: RAG 기반 보고서 생성
1. 사용자: "지식베이스에서 Q3 매출 데이터 찾아서 분기 보고서 DOCX로 만들어줘"
2. LLM: RAG 검색 도구로 관련 데이터 검색
3. LLM: 검색 결과를 기반으로 `document_generate` 도구 호출
4. LLM: python-docx 코드 생성 (목차, 요약, 상세 분석, 차트 포함)
5. 사이드카: 코드 실행 → DOCX 파일 생성 → 다운로드

### 시나리오 3: 데이터 내보내기
1. 사용자: "이 분석 결과를 엑셀로 정리해줘"
2. LLM: openpyxl 코드 생성 (시트, 헤더, 데이터, 서식)
3. 사이드카: 실행 → XLSX 파일 반환

## 구현 방향

### MCP 도구 구조
- **도구명**: `document_generate`
- **입력**: 문서 포맷, 제목, 내용 지시, (선택) RAG 검색 결과
- **출력**: 생성된 파일 다운로드 URL
- 기존 RAG 사이드카(python-sidecar)에 통합하거나, 별도 프로세스로 분리 가능

### 보안 고려사항
- 코드 실행 샌드박스: 허용 라이브러리 화이트리스트 (문서 생성 라이브러리만)
- 네트워크 접근 차단 (외부 URL fetch 불가)
- 파일 크기 제한 (예: 50MB)
- 실행 시간 제한 (예: 30초)
- 생성 파일 임시 저장 + TTL 자동 삭제 (예: 1시간)

### BE 연동
- 파일 다운로드 API: `/api/v1/documents/download/{file_id}`
- 생성 이력 저장 (테넌트별)
- 파일 스토리지: 로컬 임시 디렉토리 또는 S3

### FE 연동
- 채팅 메시지에 파일 다운로드 카드 렌더링
- 파일 미리보기 (가능한 포맷)
- 생성 진행 상태 표시

## CR-013과의 관계

CR-013에서 정의한 3단계 중:
- **Level 1 (DAG 조립)**: 기존 워크플로우로 이미 가능 — CR-018 범위 밖
- **Level 2 (변환기)**: CR-018이 "LLM 코드 생성" 접근으로 대체. 별도 변환기 개발 불필요
- **Level 3 (제품화)**: CR-018 초기 버전 이후 점진적 고도화 (템플릿, 브랜딩, 후처리 규칙 등)

CR-013은 보류 상태 유지하고, CR-018이 실제 구현을 담당한다.
