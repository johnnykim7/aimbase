# Aimbase Agent 아키텍처 결정사항 (2026-04-09)

## 배경
FlowGuard Agent가 현재 claude -p (Claude CLI)로 로컬 소스 분석 → LLM 이중 호출 비용 + 도구 제한 문제
Aimbase의 도구 체계를 SDK로 배포하여 소비앱이 Agent로 활용하는 구조로 전환

## 핵심 구조

```
[Aimbase 서버]                              [Agent (소비앱 로컬)]
  오케스트레이터 (두뇌)                         도구 실행 (팔다리)
  LLM 호출                                   aimbase-tool-sdk 사용
  프롬프트/정책/설정 (DB)                       로컬 파일 접근
  Agent 레지스트리 (주소 관리)                   MCP 서버로 대기
  MCPToolExecutor (기존)                      DB 의존성 없음
```

## 결정 사항
- Agent는 도구 실행만 한다 — LLM 호출 없음, 비용 절감
- 오케스트레이션은 Aimbase 서버가 담당 — LLM + 도구 루프는 서버에서
- 프롬프트/정책/설정은 Aimbase DB — Agent에 DB 의존성 없음
- Aimbase 내부 도구 실행은 기존 그대로 — sdk-core를 의존성으로 가져다 in-process 실행
- 원격 Agent 통신은 MCP — Aimbase MCPToolExecutor 기존 코드 활용
- 네트워크는 STUN/TURN 경유 — Agent가 STUN으로 공인주소 확인 후 Aimbase에 등록, 필요할 때만 MCP 연결
- 상시 연결 없음 — Agent는 MCP 서버로 대기, Aimbase가 필요할 때 연결 → 작업 → 종료

## aimbase-tool-sdk 구조

```
aimbase-tool-sdk
├── core  → 순수 도구 라이브러리 (Bash, Read, Grep, Glob, Edit...)
│            통신 없음, DB 없음, 프로토콜 무관
└── mcp   → MCP 서버 자동 노출 (선택 모듈)
             도구를 MCP로 감싸주는 껍데기 포함
             소비앱이 직접 만들 필요 없음
```

```
// 도구만 필요한 앱
implementation("com.platform:aimbase-tool-sdk-core")

// MCP 서버까지 필요한 앱 (Agent 용도)
implementation("com.platform:aimbase-tool-sdk-mcp")
```

## Agent 생명주기

```
Agent 기동
  → STUN 서버에 요청 → 내 공인주소 확인
  → Aimbase에 자동 등록: { user_id, public_address }
  → MCP 서버로 대기

도구 실행 필요 시
  → Aimbase가 등록된 주소로 MCP 연결 (직접 안 되면 TURN 경유)
  → 도구 루프 실행
  → 완료 후 MCP 연결 종료

Agent 종료
  → Aimbase 레지스트리에서 자동 해제
```

## 도구 실행 흐름

```
1. 소비앱 → Aimbase REST: "이 에러 분석해줘"
2. Aimbase 오케스트레이터 → LLM: "어떤 도구 쓸까?"
3. LLM: "Read(/src/Main.java) 써"
4. Aimbase → 해당 사용자 Agent 주소 조회 → MCP 연결
5. Aimbase → Agent MCP: tool_call Read → Agent가 실행 → 결과 반환
6. Aimbase → LLM: "파일 내용 이거야, 다음은?"
7. 3~6 반복 (최대 5회 루프)
8. MCP 연결 종료
9. Aimbase → 소비앱: "최종 분석 결과"
```

## 소비앱별 사용 패턴

```
① LLM 대화만 필요             → Aimbase REST API 호출 (SDK 불필요)
② 로컬 도구만 필요             → sdk-core 의존성
③ Aimbase 연동 Agent 구축     → sdk-mcp 의존성 + STUN 주소 등록
```

## 사전 조건
- STUN/TURN 서버 구축 완료 (사내 공용 인프라, 별도 구축 예정)

## 만들 것
1. aimbase-tool-sdk-core — platform-core에서 도구 구현체 추출
2. aimbase-tool-sdk-mcp — 도구를 MCP 서버로 자동 노출하는 모듈
3. Aimbase Agent 레지스트리 — Agent 자동 등록(STUN 주소)/해제/조회 API
4. Aimbase 원격 도구 실행 — 등록된 Agent 주소로 MCP 연결하여 도구 호출
5. FlowGuard Agent 측 — claude -p 제거, sdk-mcp 의존성 추가, STUN 주소 등록
