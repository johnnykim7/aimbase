# CR-036 원본 요구사항: 프롬프트 외부화 + 영문 전환 + OpenClaude 프롬프트 전수 포팅

> 작성일: 2026-04-08
> 요청자: sykim

## 배경

Aimbase의 시스템 프롬프트가 Java/Python 소스코드에 하드코딩되어 있어 다음 문제가 발생:
1. 프롬프트 수정 시 재배포 필수
2. 한글 프롬프트는 토큰 비용 2~3배
3. 런타임 A/B 테스트, 미세조정 불가
4. OpenClaude 대비 프롬프트 세분화 부족

## 요구사항

### 1. 프롬프트 외부화 인프라
- prompt_templates 별도 테이블 신규 생성 (기존 prompts 테이블과 분리)
- 3단계 폴백: Caffeine 캐시 → DB → resources/prompts/*.txt 파일
- PromptTemplateService: 조회/렌더링/캐시 관리
- REST API: CRUD + 렌더링 + 벌크 로드

### 2. 기존 하드코딩 프롬프트 DB 이관 (20~25개)
- Java 18개: AgentType(5), ContextAssemblyEngine(1), RAGService(1), LlmCallStepExecutor(5), ImageAnalysisTool(1), TranslationTool(1), ConversationSummarizer(2), MemoryAutoExtractService(1), AnthropicAdapter(1), OllamaAdapter(1)
- Python 7개: query_transformer(3), contextual_chunker(1), evaluator(3)
- 한글 프롬프트 전체 영문 전환 (평가 프롬프트 포함)

### 3. OpenClaude 프롬프트 전수 포팅
- OpenClaude 소스: /Users/sykim/Downloads/openclaude-main/
- Core 시스템 프롬프트 5개 (prompts.ts 914줄, system.ts 등)
- Tool별 프롬프트 36개 (각 tools/*/prompt.ts)
- 서비스 프롬프트 6개 (extractMemories, SessionMemory, MagicDocs, compact 등)
- 동적/설정 프롬프트 20+개 (언어, MCP, Git, 출력스타일 등)
- 총 70+개, ~5,450줄 규모

### 4. Python 사이드카 연동
- 시작 시 BE API 한번 호출하여 벌크 로드 + 메모리 캐시
- 실패 시 로컬 파일 폴백

### 5. FE 관리 화면
- 프롬프트 템플릿 목록/편집/카테고리 필터
- 테스트 렌더링 패널 (변수 입력 → 프리뷰 + 토큰 추정)
- is_system=true 프롬프트는 편집 가능, 삭제 불가
