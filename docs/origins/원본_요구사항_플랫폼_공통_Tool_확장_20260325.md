## 배경

Aimbase는 MCP Tool을 추가할수록 LLM 오케스트레이터의 능력이 확장되는 구조.
현재 27개 Tool이 등록되어 있으나, 범용 플랫폼으로서 부족한 Tool들이 있음.

핵심 원칙:
- **Tool = LLM이 의도를 가지고 호출하는 단위**. 단순 파일 I/O(`read_file`, `save_file`)는 Tool이 아님
- Aimbase = 범용 도구 플랫폼, 소비앱 = 도메인 도구 자체 등록
- 소비앱은 Extension 인터페이스 또는 MCP 서버로 도메인 Tool 자체 등록

## 추가 요청 Tool (12개)

### 문서 읽기 (4개)
1. **read_pptx** — PPTX 전체를 구조화된 JSON/HTML로 일괄 변환 (텍스트+레이아웃+이미지). 230페이지도 1회 호출로 처리
2. **read_docx** — DOCX → 구조화된 JSON/HTML (헤더/푸터/표/이미지 포함)
3. **read_excel** — XLSX/CSV → JSON (시트별 데이터)
4. **read_pdf** — PDF → 텍스트/구조 추출 (OCR 포함 가능)

### 변환 (1개)
5. **convert_format** — 포맷 간 변환 (DOCX↔PDF↔PPTX↔HTML). LibreOffice headless 활용

### 실행/시각화 (2개)
6. **run_python** — 샌드박스 Python 코드 실행 (데이터 가공, 계산, 파일 생성)
7. **render_chart** — 데이터 → 차트 이미지 (matplotlib/plotly)

### 웹/외부 (2개)
8. **web_search** — 웹 검색 결과 반환
9. **web_fetch** — URL → 정제된 콘텐츠

### 통신 (1개)
10. **send_notification** — 이메일/슬랙/웹훅 발송

### 분석/번역 (2개)
11. **analyze_image** — 이미지 → 설명/데이터 추출 (멀티모달 LLM Vision)
12. **translate_text** — 다국어 번역 (LLM 체인)

## 추가 요청: Tool 목록 API

- 워크플로우 편집기에서 Tool을 선택하려면 `GET /api/v1/tools` API 필요
- `ToolRegistry.getToolDefs()`는 이미 존재, Controller만 추가
- 워크플로우 TOOL_CALL 스텝에서 Tool 이름으로 호출하는 구조 (`ToolCallStepExecutor`)
