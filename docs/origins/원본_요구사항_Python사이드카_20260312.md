## 원문

### 배경

현재 Aimbase은 Spring Boot 3.4.2 (Java 21) 기반 모놀리식 백엔드로 구현되어 있다. Python AI 생태계(LangChain, LlamaIndex, RAGAS, Presidio 등)가 압도적으로 풍부하여, AI 특화 기능은 Python 사이드카로 분리하는 하이브리드 아키텍처를 도입한다.

핵심 원칙:
- Spring은 엔터프라이즈 오케스트레이션(멀티테넌트, 정책, 워크플로우, RBAC 등)을 담당
- Python은 AI 특화 기능(RAG 고도화, 평가, PII 탐지 등)을 담당
- 연동은 MCP(Model Context Protocol) 기반 직접 호출 (LLM 경유 불필요)
- 기존 Spring MCP Client 인프라를 활용하여 Spring 코드 변경 최소화

### Python MCP Server 구조

```
MCP Server 1: RAG Pipeline
  - 문서 파싱 (Unstructured)          ← PRD-052 이관
  - 시맨틱 청킹 (LlamaIndex)          ← PY-001 신규
  - 로컬 임베딩 (sentence-transformers) ← PY-002 신규
  - 하이브리드 검색 + 리랭킹            ← PRD-053 이관
  - 쿼리 변환 (HyDE)                  ← PY-005 신규

MCP Server 2: Evaluation
  - RAG 품질 평가 (RAGAS)             ← PY-006 신규
  - LLM 출력 평가 (DeepEval)          ← PY-007 신규
  - 프롬프트 회귀 테스트               ← PY-008 신규

MCP Server 3: Safety
  - PII 탐지/마스킹 (Presidio)        ← PRD-095 이관
  - 출력 가드레일                      ← PY-010 신규

MCP Server 4: Agent (향후)
  - 고급 추론 체인 (LangGraph)         ← PY-011 신규
```

### 연동 방식

- Spring → Python: MCP Client → MCP Server (직접 호출, LLM 경유 불필요)
- MCP는 표준화된 JSON-RPC 프로토콜로, 순수 서비스 간 통신으로 사용
- 기존 Spring MCP Client 인프라 재활용
