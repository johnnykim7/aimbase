# CR-012 요구사항 원본 — 로컬 임베딩 전환 (BGE-M3 기본화)

> 일시: 2026-03-22
> 요청자: sykim

## 대화 요약

### 배경
- Aimbase 수동 테스트 환경 구축 중, OpenAI API key 없이는 백엔드 기동 불가 문제 발견
- `EmbeddingService` → `OpenAiEmbeddingModel` 빈이 필수 의존이라 API key 없으면 Spring 컨텍스트 실패
- Python 사이드카에 이미 sentence-transformers 기반 로컬 임베딩(KoSimCSE) 인프라가 존재

### 사용자 질문 및 결정
1. "임베딩이 OpenAI가 가장 강력한가?" → **아님**. MTEB 벤치마크 기준:
   - Voyage-3-large, Cohere embed-v4가 유료 최상위
   - BGE-M3 (무료 오픈소스)가 OpenAI text-embedding-3-small보다 성능 우위
   - 현재 프로젝트의 text-embedding-3-small은 유료이면서 중간 성능
2. "무료로 직접 만들면 어떨까?" → **BGE-M3 채택 결정**
   - 다국어(한국어 포함) 오픈소스 최상위
   - 1024차원, ~2.2GB
   - Python 사이드카에서 sentence-transformers로 바로 사용 가능
3. "인터페이스는 선택적으로 남겨두고" → **OpenAI도 옵션으로 유지**, 기본값만 BGE-M3로 변경

### 변경 범위
- Python 사이드카: 기본 모델 KoSimCSE(768d) → BGE-M3(1024d)
- DB: vector(1536) 고정 → 가변 차원으로 변경 (마이그레이션)
- BE: EmbeddingService Optional화 (API key 없이 기동 가능, MCP 사이드카 경유)
- BE: 임베딩 제공자 선택 설정 (OpenAI / 로컬 MCP)
