## 한줄 소개

**여러 AI 모델(ChatGPT, Claude 등)을 하나의 통합 플랫폼에서 관리하고, 보안 정책을 적용하며, 자동화 워크플로우를 구축할 수 있는 멀티테넌트 LLM 오케스트레이션 플랫폼입니다.**

3. 보안 정책 설정             │                            │
   (비용 제한, 승인 필요 등)   │                            │
   │                          │                            │
   ▼                          ▼                            │
4. 사용자/역할 생성 ───────> 로그인                        │

## 주요 개념 설명

| 개념 | 쉬운 설명 | 실생활 비유 |
|------|-----------|------------|
| **테넌트** | 독립된 조직 공간. 데이터가 완전히 분리됨 | 같은 건물의 다른 사무실 |
| **연결(Connection)** | 외부 AI 서비스와의 접속 정보 | 전화번호부에 등록된 연락처 |
| **정책(Policy)** | AI 사용에 대한 규칙. 비용 제한, 승인 필요 등 | 회사의 결재 규정 |
| **프롬프트** | AI에게 보내는 질문/지시 템플릿 | 자주 쓰는 이메일 양식 |
| **워크플로우** | 여러 작업을 순서대로 자동 실행하는 흐름 | 결재 라인 (작성→검토→승인→실행) |
| **RAG (지식 검색)** | 문서를 학습시켜 AI가 정확한 답변을 할 수 있게 함 | 참고 자료를 읽고 답변하는 것 |
| **MCP 서버** | 외부 도구를 AI에 연결하는 표준 방법 | USB로 외부 장치를 컴퓨터에 연결 |
| **스키마** | AI 응답의 형식을 지정하는 규칙 | 양식에 맞춰 작성하라는 지시 |
| **역할(Role)** | 사용자가 할 수 있는 일의 범위 | 직급별 권한 (사원/팀장/부서장) |
| **API 키** | 프로그램이 플랫폼에 접속하기 위한 비밀 열쇠 | 사원증 |
| **구독(Subscription)** | 테넌트가 사용할 수 있는 자원의 한도 | 데이터 요금제 (월 100만 토큰) |

## 주요 기능 흐름

### AI 채팅 흐름

```
사용자 질문 입력
    │
    ▼
세션 이력 불러오기 (이전 대화 기억)
    │
    ▼
정책 평가 (비용 제한 확인, 보안 검사)
    │
    ├── 거부 → "정책에 의해 차단됨" 메시지
    │
    ▼
AI 모델 선택 (라우팅 규칙에 따라)
    │
    ▼
AI에게 질문 전달
    │
    ├── AI가 도구 필요 시 → 도구 실행 → 결과로 다시 질문
    │
    ▼
AI 응답 수신
    │
    ▼
사용량 기록 (토큰 수, 비용)
    │
    ▼
응답 반환
```

### 워크플로우 흐름 (예: 고객 문의 자동 처리)

```
1. [트리거] 문의 접수
       │
       ▼
2. [AI 호출] 문의 내용 분류 (주문/배송/환불/기타)
       │
       ▼
3. [조건 분기] 환불 요청인가?
       ├── Yes → [승인 대기] 관리자 승인 필요
       │            ├── 승인 → [액션] DB에 환불 처리 기록
       │            │          [알림] Slack으로 처리 완료 통보
       │            └── 거부 → [알림] 거부 사유 통보
       │
       └── No → [AI 호출] 자동 답변 생성
                    │
                    ▼
                 [알림] 답변 발송
```

## FAQ

**Q1. AI 모델은 어떤 것을 지원하나요?**
A. OpenAI (GPT-4o 등), Anthropic Claude (Claude Sonnet 등), Ollama (로컬 AI 모델)를 지원합니다. 연결 관리에서 원하는 모델을 등록할 수 있습니다.

**Q2. 데이터 보안은 어떻게 보장되나요?**
A. 각 조직(테넌트)은 완전히 독립된 데이터베이스를 사용합니다. 다른 조직의 데이터를 볼 수 없으며, 정책을 통해 민감정보 마스킹(PII 마스킹)도 가능합니다.

**Q3. 비용 관리는 어떻게 하나요?**
A. 정책 기능으로 일일/월간 토큰 사용 한도를 설정할 수 있습니다. 대시보드에서 실시간 비용을 확인하고, 모니터링에서 모델별 비용을 비교할 수 있습니다.

**Q4. 워크플로우에서 승인이 필요한 작업은 어떻게 처리하나요?**
A. 워크플로우에 "승인" 단계를 추가하면, 해당 단계에서 자동으로 멈추고 관리자에게 알림이 갑니다. 관리자가 승인하면 다음 단계가 자동 진행됩니다.

**Q5. 기존 문서를 AI가 참고하게 하려면 어떻게 하나요?**
A. 지식 관리(Knowledge)에서 문서를 업로드하고 동기화하면, AI가 채팅 시 해당 문서 내용을 참고하여 답변합니다. PDF, Office 문서 등을 지원합니다.

**Q6. 여러 AI 모델을 동시에 사용할 수 있나요?**
A. 네. 라우팅 규칙을 설정하면 질문 유형에 따라 자동으로 적합한 모델이 선택됩니다. 비용 최적화, 응답속도 우선 등 전략을 설정할 수 있습니다.
